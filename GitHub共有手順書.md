# ローカルフォルダをGitHubに登録（プッシュ）する手順書

この手順書は、お手元のパソコン内にあるファイル（ローカルフォルダ）を、インターネット上のGitHub上に新しく登録（共有・バックアップ）する際の一連の流れをまとめたものです。

## 全体の流れ
1. Gitの初期設定（名前とメールアドレスの登録）
2. ローカルフォルダをGitの管理下にする（コミット）
3. GitHub上で「空の」リポジトリ（受け皿）を作成する
4. ローカルとGitHubを紐付けてアップロードする

---

## 1. Gitの初期設定を行う
Gitを初めて使う場合や、設定が漏れている場合は以下のコマンドを実行してご自身の情報を設定します。この情報は「誰がファイルを保存したか」を記録するために必要です。

ターミナル（コマンドプロンプト、PowerShell、VSCodeのターミナル等）を開いて以下を実行します。

```bash
git config --global user.name "ご自身のGitHubユーザー名"
git config --global user.email "ご自身のGitHubメールアドレス"
```

## 2. ローカルフォルダをGitの管理下にする
登録したいフォルダに移動し、ファイルをローカルのGitリポジトリ（手元の履歴管理システム）に保存（コミット）します。

```bash
# 1. フォルダの初期化
git init

# (任意) gitignoreファイルの作成
# プログラムをアップロードする際、キャッシュなどの不要なファイルを除外するため、
# 事前に `.gitignore` というファイルを作成しておくと便利です。

# 2. 現在のブランチ名を「main」に統一する
git branch -M main

# 3. フォルダ内の全ての変更やファイルを追加する
git add .

# 4. バージョンの保存（コミット）を行う
git commit -m "Initial commit（最初のコミット）"
```
これで手元のパソコン内での履歴保存が完了しました。

## 3. GitHub上で「空の」リポジトリを作成する
次に、ファイルのアップロード先となる「受け皿」をGitHub上に作ります。

1. [GitHub](https://github.com/) にブラウザでログインします。
2. 画面右上などにある「**+**」アイコンをクリックし、「**New repository**」を選択します。
3. `Repository name` に好きなプロジェクト名を入力します。
4. **【重要】** 画面下部の「Add a README file」や「Add .gitignore」のチェックボックスは**すべて外したまま（空のまま）**にしてください。
   （※ここにチェックを入れると、手元のファイルと競合してエラーになる原因となります）
5. 「**Create repository**」の緑色のボタンをクリックします。
6. 作成されると `https://github.com/アカウント名/リポジトリ名.git` というURLが表示されるので、それをコピーします。

## 4. ローカルとGitHubを紐付けてアップロードする（Push）
最後に、手順2で保存した手元の状態を、手順3で作ったGitHubの受け皿に打ち上げます。

ターミナルに戻り、以下のコマンドを実行します。

```bash
# 1. ローカルのGitにリモート（アップロード先）のURLを紐付ける
# （※「コピーしたURL」の部分には、実際の手順3のURLを入れてください）
git remote add origin コピーしたURL

# 2. GitHubへファイルをプッシュ（アップロード）する
git push -u origin main
```

### 認証について（初回のみ）
上記の `git push` コマンドを実行した際、認証を求めるポップアップ画面（Git Credential Managerなど）が表示されることがあります。
その場合は「Sign in with your browser」などをクリックしてブラウザに遷移し、GitHubのアカウントへのアクセスを許可してください。

認証が完了すると自動的にプッシュが進み、GitHubのページを更新するとファイルが無事にアップロードされていることが確認できます。

---
以上で作業フォルダのGitHubへの登録は完了です！
次回以降、ファイルを更新してアップロードしたい場合は、以下の手順だけでOKです。

```bash
git add .
git commit -m "変更内容のメモ"
git push
```
